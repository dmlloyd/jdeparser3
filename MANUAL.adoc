[[introduction]]
== Introduction

JDeparser 3 is a Java source code generation library.
Rather than concatenating strings or manipulating template files, JDeparser provides a type-safe API for constructing syntactically correct Java source code.
The library is designed for two primary use cases:

* *Annotation processors* that generate source files during compilation
* *Build-time code generators* that produce Java source as part of a build pipeline

=== Architecture Overview

The API is organized into three area:

Setup:: Configure the output destination (`Filer`), code formatting (`FormatPreferences`), and target Java version (`SourceVersion`), then create a `Sources` collection.

Creators:: Define the shape of your code -- classes, methods, fields, constructors -- using _creator_ callbacks.
Each structural construct is built through a `Consumer<XxxCreator>` callback where the creator object is valid only during the callback (the _borrow pattern_).

Values (Types and Expressions):: `Type` and `Expr` are composable value objects that represent Java types and expressions.
Unlike creators, they can be freely stored and reused.

=== Maven Coordinates

[source,xml]
----
<dependency>
    <groupId>io.smallrye.jdeparser</groupId>
    <artifactId>jdeparser</artifactId>
    <version>3.0-SNAPSHOT</version>
</dependency>
----

=== Java Module

The Java module for this library is named `io.smallrye.jdeparser`.

=== Requirements

JDeparser 3 requires JDK 17 or later at compile time.
The generated source code can target any Java version from 8 to 26, controlled by the `SourceVersion` parameter.

[[quick-start]]
== Quick Start

This chapter walks through a complete, minimal example that generates a Java source file.
The program creates a `Person` class with a `name` field, a constructor, and a getter method.

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class QuickStart {
    public static void main(String[] args) throws IOException {
        // 1. Configure output: write to the "generated-sources" directory
        Filer filer = Filer.newInstance(Path.of("generated-sources"));

        // 2. Use default formatting and target Java 17
        FormatPreferences format = FormatPreferences.defaults();
        SourceVersion version = SourceVersion.JAVA_17;

        // 3. Create the source file collection
        Sources sources = JDeparser.createSources(filer, format, version);

        // 4. Define a source file
        sources.createSourceFile("com.example", "Person", sf -> { // <1>
            sf.class_("Person", cc -> { // <2>
                cc.public_();
                cc.final_();

                // Field: private final String name;
                cc.field("name", fc -> { // <3>
                    fc.private_();
                    fc.final_();
                    fc.type(Type.STRING);
                });

                // Constructor: public Person(String name) { this.name = name; }
                cc.constructor(con -> { // <4>
                    con.public_();
                    con.param("name", Type.STRING);
                    con.body(body -> {
                        body.emit(
                            Expr.THIS.field("name").assign(Expr.$v("name"))
                        );
                    });
                });

                // Getter: public String getName() { return name; }
                cc.method("getName", mc -> { // <5>
                    mc.public_();
                    mc.returning(Type.STRING);
                    mc.body(body -> {
                        body.return_(Expr.$v("name"));
                    });
                });
            });
        });

        // 5. Write all source files to disk
        sources.writeSources(); // <6>
    }
}
----
<1> Create a source file in package `com.example` named `Person.java`.
<2> Define a `public final class Person` inside the file.
<3> Define the `name` field with type, access, and modifier.
<4> Define the constructor with a parameter and body.
<5> Define the `getName()` getter method.
<6> Flush all defined source files to the `Filer`.

Running this program produces `generated-sources/com/example/Person.java`:

[source,java]
----
package com.example;

public final class Person {

    private final String name;

    public Person(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}
----


[[borrow-pattern]]
== The Borrow Pattern

The borrow pattern is the central design principle of JDeparser 3.
Every structural construct -- classes, methods, blocks, switch cases, try statements -- is built through a `Consumer` callback that receives a _creator_ object.
The creator is valid *only* during the callback; it is "borrowed" for the duration of the lambda.

=== Why Callbacks?

Consider the alternative: a builder API that returns the builder from each method.
With that approach, nothing prevents you from interleaving calls to unrelated builders, or storing a builder and using it after the structure it represents has already been finalized.
The borrow pattern makes such misuse impossible at the API level.

=== Creator Lifecycle

When you call a method like `cc.method("foo", mc -> { ... })`:

1. A `MethodCreator` is created and becomes _active_.
2. The parent `ClassCreator` becomes _inactive_ for the duration of the callback.
3. Your lambda runs with the active `MethodCreator`.
4. When the lambda returns, the `MethodCreator` is _finalized_ and becomes permanently inactive.
5. The parent `ClassCreator` becomes active again.

Any attempt to use a creator after its callback has returned will throw `IllegalStateException`.

=== Example: Safety in Action

[source,java]
----
import java.io.IOException;
import java.io.StringWriter;
import java.nio.file.Path;

import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.creator.MethodCreator;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class BorrowPatternDemo {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        // Capturing a creator reference outside its callback
        MethodCreator[] captured = new MethodCreator[1];

        sources.createSourceFile("com.example", "Demo", sf -> {
            sf.class_("Demo", cc -> {
                cc.public_();
                cc.method("greet", mc -> {
                    mc.public_();
                    mc.returning(Type.VOID);
                    captured[0] = mc; // capture the creator reference
                });
            });
        });

        // The callback is over -- the creator is finalized
        try {
            captured[0].returning(Type.STRING); // throws IllegalStateException!
        } catch (IllegalStateException e) {
            System.out.println("Caught: " + e.getMessage());
        }
    }
}
----

TIP: Think of the callback as a scope: everything you need to configure must happen inside it.
If you find yourself wanting to store a creator for later use, restructure your code so all configuration happens within the callback.


[[setup]]
== Setup: Filer, Format, and Source Version

Every code generation session begins with three configuration objects passed to `JDeparser.createSources()`:

[source,java]
----
Sources sources = JDeparser.createSources(filer, preferences, sourceVersion);
----

=== Filer

The `Filer` interface abstracts the output destination.
It has a single method:

[source,java]
----
Writer openWriter(String packageName, String fileName) throws IOException;
----

Three factory methods cover the common cases:

[cols="1,2"]
|===
| Factory Method | Use Case

| `Filer.newInstance(Path outputDir)`
| Write to a directory on the filesystem (NIO).

| `Filer.newInstance(File outputDir)`
| Write to a directory on the filesystem (legacy IO).

| `Filer.newInstance(javax.annotation.processing.Filer filer)`
| Write via the annotation processing API.
|===

You can also implement the `Filer` interface directly for custom output targets.
For example, writing to an in-memory `StringWriter` for testing:

[source,java]
----
import java.io.StringWriter;

StringWriter sw = new StringWriter();
Filer testFiler = (packageName, fileName) -> sw;
// ... generate code ...
sources.writeSources();
String generatedCode = sw.toString();
----

=== FormatPreferences

`FormatPreferences` controls how the generated source code is formatted: indentation, spacing around operators, line wrapping, and more.

For most use cases, the defaults are sufficient:

[source,java]
----
FormatPreferences format = FormatPreferences.defaults();
----

To customize, use the builder:

[source,java]
----
import io.smallrye.jdeparser.format.FormatPreferences;
import io.smallrye.jdeparser.format.FormatPreferences.*;

FormatPreferences format = FormatPreferences.builder()
    .lineLength(80)
    .indent(Indentation.MEMBERS_TOP_LEVEL, 2)
    .indent(Indentation.LINE, 2)
    .space(Space.BEFORE_BRACE_CLASS, SpaceType.NEWLINE)  // Allman brace style
    .space(Space.BEFORE_BRACE_METHOD, SpaceType.NEWLINE)
    .wrapMode(Wrapping.PARAMETER_LIST, WrappingMode.ALWAYS_WRAP)
    .addOption(Opt.ENUM_TRAILING_COMMA)
    .build();
----

The builder also supports loading configuration from a `Properties` object:

[source,java]
----
Properties props = new Properties();
props.setProperty("line-length", "100");
props.setProperty("indent.members-top-level", "2");
props.setProperty("space.before-brace-class", "NEWLINE");

FormatPreferences format = FormatPreferences.builder()
    .fromProperties(props)
    .build();
----

See <<formatting>> for a complete reference of all formatting options.

=== SourceVersion

`SourceVersion` determines which language features are permitted.
If you attempt to use a feature that requires a higher Java version than your target, the library throws `IllegalStateException` at construction time.

[source,java]
----
SourceVersion version = SourceVersion.JAVA_17;

// Check feature support
boolean hasRecords = version.supports(LanguageFeature.RECORDS); // true (Java 16+)
boolean hasPatternSwitch = version.supports(LanguageFeature.PATTERN_MATCHING_SWITCH); // false (Java 21+)
----

Available versions range from `JAVA_8` through `JAVA_26`.
See <<source-version-features>> for the complete feature compatibility table.

=== Complete Setup Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;
import io.smallrye.jdeparser.format.FormatPreferences.*;

public class SetupExample {
    public static void main(String[] args) throws IOException {
        // Custom formatting: 2-space indent, 100-column lines
        FormatPreferences format = FormatPreferences.builder()
            .lineLength(100)
            .indent(Indentation.MEMBERS_TOP_LEVEL, 2)
            .indent(Indentation.LINE, 2)
            .indent(Indentation.LINE_CONTINUATION, 2)
            .build();

        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(filer, format, SourceVersion.JAVA_17);

        sources.createSourceFile("com.example", "Config", sf -> {
            sf.class_("Config", cc -> {
                cc.public_();
                cc.field("DEBUG", fc -> {
                    fc.public_();
                    fc.static_();
                    fc.final_();
                    fc.type(Type.BOOLEAN);
                    fc.init(io.smallrye.jdeparser.Expr.FALSE);
                });
            });
        });

        sources.writeSources();
    }
}
----


[[types]]
== Types

The `Type` interface represents Java types in generated source code.
Types are immutable, composable values: methods such as `array()` and `typeArg()` return new `Type` instances rather than modifying the original.

=== Primitive Types

Predefined constants are available for all nine primitive types (including `void`):

[source,java]
----
Type.VOID     // void
Type.BOOLEAN  // boolean
Type.BYTE     // byte
Type.SHORT    // short
Type.INT      // int
Type.LONG     // long
Type.FLOAT    // float
Type.DOUBLE   // double
Type.CHAR     // char
----

=== Reference Types

Reference types can be created from `Class` objects, fully qualified names, or annotation processing `TypeMirror` objects:

[source,java]
----
// From a Class object
Type listType = Type.of(List.class);          // java.util.List

// From a fully qualified name
Type myType = Type.named("com.example.MyClass");

// From a TypeMirror (in annotation processors)
Type mirrorType = Type.of(typeMirror);
----

Two commonly used reference types have predefined constants:

[source,java]
----
Type.STRING   // java.lang.String
Type.OBJECT   // java.lang.Object
----

=== Array Types

Call `array()` to create an array type.
Chain multiple calls for multidimensional arrays:

[source,java]
----
Type intArray = Type.INT.array();                // int[]
Type stringMatrix = Type.STRING.array().array();  // String[][]
----

=== Parameterized Types

Use `typeArg()` to apply type arguments:

[source,java]
----
Type listOfString = Type.of(List.class).typeArg(Type.STRING);
// List<String>

Type mapType = Type.named("java.util.Map").typeArg(Type.STRING, Type.INT.box());
// Map<String, Integer>
----

=== Wildcard Types

The unbounded wildcard is a predefined constant.
Bounded wildcards are created from the bound type:

[source,java]
----
Type wildcard = Type.WILDCARD;                          // ?
Type extendsNumber = Type.named("java.lang.Number").wildcardExtends();  // ? extends Number
Type superString = Type.STRING.wildcardSuper();          // ? super String
----

=== Nested Types

Use `nestedType()` for inner or nested types:

[source,java]
----
Type mapEntry = Type.named("java.util.Map").nestedType("Entry");
// Map.Entry

Type mapEntryGeneric = Type.named("java.util.Map").nestedType("Entry")
    .typeArg(Type.STRING, Type.OBJECT);
// Map.Entry<String, Object>
----

=== Intersection Types

Use `Type.allOf()` to create intersection types (used in type bounds and casts):

[source,java]
----
Type serializableComparable = Type.allOf(
    Type.named("java.io.Serializable"),
    Type.named("java.lang.Comparable")
);
// Serializable & Comparable
----

=== Type Conversions

[source,java]
----
Type boxedInt = Type.INT.box();           // java.lang.Integer
Type unboxedInt = boxedInt.unbox();       // int
Type erased = listOfString.erasure();     // java.util.List (strips type args)
----

=== Types as Expression Roots

Types serve as entry points for static member access, constructor calls, class literals, and method references:

[source,java]
----
// Static field access
Var out = Type.named("java.lang.System").field("out");  // System.out

// Static method call
Expr parsed = Type.INT.box().call("parseInt", Expr.str("42"));  // Integer.parseInt("42")

// Class literal
Expr classLit = Type.STRING.class_();  // String.class

// Constructor call
Expr newList = Type.named("java.util.ArrayList").new_();  // new ArrayList()

// Array creation with initializer
Expr array = Type.INT.array().newArrayInit(Expr.decimal(1), Expr.decimal(2));
// new int[] { 1, 2 }

// Method reference
Expr ref = Type.STRING.methodRef("valueOf");  // String::valueOf

// Qualified this/super
Expr qualThis = Type.named("com.example.Outer").this_();   // Outer.this
Expr qualSuper = Type.named("com.example.Outer").super_(); // Outer.super
----

=== Complete Types Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;
import java.util.List;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class TypesExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        Type listType = Type.of(List.class);
        Type mapType = Type.named("java.util.Map");
        Type numberType = Type.named("java.lang.Number");

        sources.createSourceFile("com.example", "TypeShowcase", sf -> {
            sf.import_(List.class);
            sf.import_(Type.named("java.util.Map"));
            sf.import_(Type.named("java.util.ArrayList"));

            sf.class_("TypeShowcase", cc -> {
                cc.public_();

                // List<Map<String, ? extends Number>> complexField;
                cc.field("complexField", fc -> {
                    fc.type(listType.typeArg(
                        mapType.typeArg(Type.STRING, numberType.wildcardExtends())
                    ));
                });

                // int[][] matrix;
                cc.field("matrix", fc -> {
                    fc.type(Type.INT.array().array());
                });

                // Map.Entry<String, Object> entry;
                cc.field("entry", fc -> {
                    fc.type(mapType.nestedType("Entry")
                        .typeArg(Type.STRING, Type.OBJECT));
                });

                // static method demonstrating type expressions
                cc.method("demo", mc -> {
                    mc.public_();
                    mc.static_();
                    mc.returning(Type.VOID);
                    mc.body(body -> {
                        // List<String> names = new ArrayList<>();
                        body.var(listType.typeArg(Type.STRING), "names",
                            Type.named("java.util.ArrayList").new_());

                        // System.out.println(String.class);
                        body.emit(
                            Type.named("java.lang.System").field("out")
                                .call("println", Type.STRING.class_())
                        );

                        // Integer.parseInt("42")
                        body.var(Type.INT, "value",
                            Type.INT.box().call("parseInt", Expr.str("42")));
                    });
                });
            });
        });

        sources.writeSources();
    }
}
----


[[expressions]]
== Expressions

The `Expr` interface represents Java expressions.
Like types, expressions are immutable, composable values.
The `Var` interface extends `Expr` to represent assignable targets (variables, fields, array elements).

=== Literals

==== Numeric Literals

[source,java]
----
Expr.decimal(42)       // 42
Expr.decimal(42L)      // 42L
Expr.decimal(3.14f)    // 3.14f
Expr.decimal(3.14)     // 3.14

Expr.hex(0xFF)         // 0xff
Expr.hex(0xFFL)        // 0xffL

Expr.octal(077)        // 077
Expr.octal(077L)       // 077L

Expr.binary(0b1010)    // 0b1010
Expr.binary(0b1010L)   // 0b1010L
----

==== String and Character Literals

[source,java]
----
Expr.str("hello")         // "hello"
Expr.textBlock("line1\nline2\n")  // """
                                   //     line1
                                   //     line2
                                   //     """
Expr.ch('A')              // 'A'
Expr.ch('\n')             // '\n'
----

NOTE: `textBlock()` requires `SourceVersion.JAVA_15` or later.

==== Predefined Constants

[source,java]
----
Expr.TRUE    // true
Expr.FALSE   // false
Expr.NULL    // null
Expr.THIS    // this
Expr.SUPER   // super
Expr.ZERO    // 0
Expr.ONE     // 1
----

=== Variables

Use `Expr.$v()` to reference a local variable or parameter by name:

[source,java]
----
Var x = Expr.$v("x");   // x
Var args = Expr.$v("args"); // args
----

The returned `Var` supports all `Expr` operations plus assignment.

=== Arithmetic Operations

[source,java]
----
Expr a = Expr.$v("a");
Expr b = Expr.$v("b");

a.add(b)    // a + b
a.sub(b)    // a - b
a.mul(b)    // a * b
a.div(b)    // a / b
a.mod(b)    // a % b
a.neg()     // -a
a.pos()     // +a
----

Expressions compose naturally and the library handles operator precedence and parenthesization:

[source,java]
----
// (a + b) * c  -- parentheses inserted automatically where needed
Expr result = a.add(b).mul(Expr.$v("c"));
----

=== Bitwise and Shift Operations

[source,java]
----
a.bitAnd(b)   // a & b
a.bitOr(b)    // a | b
a.bitXor(b)   // a ^ b
a.comp()      // ~a

a.shl(b)      // a << b
a.shr(b)      // a >> b
a.ushr(b)     // a >>> b
----

=== Relational and Equality Operations

[source,java]
----
a.eq(b)    // a == b
a.ne(b)    // a != b
a.lt(b)    // a < b
a.gt(b)    // a > b
a.le(b)    // a <= b
a.ge(b)    // a >= b
----

=== Logical Operations

[source,java]
----
a.and(b)   // a && b
a.or(b)    // a || b
a.not()    // !a
----

=== Type Operations

[source,java]
----
// Cast
Expr castExpr = a.cast(Type.STRING);  // (String) a

// instanceof
Expr test = a.instanceof_(Type.STRING);  // a instanceof String

// Pattern matching instanceof (Java 16+)
Expr pattern = a.instanceof_(Type.STRING, "s");  // a instanceof String s
----

=== Conditional (Ternary)

[source,java]
----
// condition ? trueExpr : falseExpr
Expr ternary = a.gt(b).cond(a, b);  // a > b ? a : b
----

=== Grouping

Use `paren()` to explicitly parenthesize an expression:

[source,java]
----
Expr grouped = a.add(b).paren();  // (a + b)
----

NOTE: The library automatically inserts parentheses based on operator precedence.
Use `paren()` only when you want explicit grouping beyond what precedence requires.

=== Member Access

[source,java]
----
Expr obj = Expr.$v("obj");

// Field access
Var field = obj.field("name");          // obj.name

// Method call
Expr call = obj.call("toString");       // obj.toString()
Expr callArgs = obj.call("equals", b);  // obj.equals(b)

// Array element access
Var element = obj.idx(Expr.ZERO);       // obj[0]
----

=== Increment and Decrement

[source,java]
----
Var i = Expr.$v("i");

i.inc()          // i++ (postfix)
i.dec()          // i-- (postfix)
Expr.inc(i)      // ++i (prefix)
Expr.dec(i)      // --i (prefix)
----

=== Assignments

The `Var` interface provides simple and compound assignment operators:

[source,java]
----
Var x = Expr.$v("x");

x.assign(Expr.ZERO)        // x = 0
x.addAssign(Expr.ONE)       // x += 1
x.subAssign(Expr.ONE)       // x -= 1
x.mulAssign(Expr.decimal(2))   // x *= 2
x.divAssign(Expr.decimal(2))   // x /= 2
x.modAssign(Expr.decimal(3))   // x %= 3
x.bitAndAssign(Expr.hex(0xFF)) // x &= 0xff
x.bitOrAssign(Expr.hex(0x01))  // x |= 0x1
x.bitXorAssign(Expr.hex(0xFF)) // x ^= 0xff
x.shlAssign(Expr.decimal(2))   // x <<= 2
x.shrAssign(Expr.decimal(2))   // x >>= 2
x.ushrAssign(Expr.decimal(2))  // x >>>= 2
----

=== Method References

[source,java]
----
// Instance method reference: obj::method
Expr ref1 = Expr.$v("obj").methodRef("toString");  // obj::toString

// Type method reference: Type::method
Expr ref2 = Type.STRING.methodRef("valueOf");       // String::valueOf

// Constructor reference: Type::new
Expr ref3 = Type.named("java.util.ArrayList").methodRef("new"); // ArrayList::new
----

=== Lambda Expressions

==== Expression-Body Lambdas

[source,java]
----
// Single parameter: x -> x * 2
Expr lambda1 = Expr.lambda("x", Expr.$v("x").mul(Expr.decimal(2)));

// Multiple parameters: (a, b) -> a + b
Expr lambda2 = Expr.lambda(List.of("a", "b"),
    Expr.$v("a").add(Expr.$v("b")));
----

==== Block-Body Lambdas

Block-body lambdas require a `SourceVersion` parameter:

[source,java]
----
// x -> { return x * 2; }
Expr lambda3 = Expr.lambda(SourceVersion.JAVA_17, "x", body -> {
    body.return_(Expr.$v("x").mul(Expr.decimal(2)));
});

// (a, b) -> { ... }
Expr lambda4 = Expr.lambda(SourceVersion.JAVA_17, List.of("a", "b"), body -> {
    body.var(Type.INT, "sum", Expr.$v("a").add(Expr.$v("b")));
    body.return_(Expr.$v("sum"));
});
----

==== Typed Lambdas

For lambdas with explicitly typed parameters, use `lambdaTyped()`:

[source,java]
----
import io.smallrye.jdeparser.impl.LambdaExpr.LambdaParam;

// (String s, int i) -> s.length() + i
Expr typed = Expr.lambdaTyped(
    List.of(
        new LambdaParam("s", Type.STRING),
        new LambdaParam("i", Type.INT)
    ),
    Expr.$v("s").call("length").add(Expr.$v("i"))
);
----

=== Switch Expressions

Switch expressions (Java 14+) are created from an expression:

[source,java]
----
Expr switchExpr = Expr.$v("status").switch_(SourceVersion.JAVA_17, sw -> {
    sw.case_(Expr.str("OK"), b -> {
        b.yield_(Expr.decimal(200));
    });
    sw.case_(Expr.str("NOT_FOUND"), b -> {
        b.yield_(Expr.decimal(404));
    });
    sw.default_(b -> {
        b.yield_(Expr.decimal(500));
    });
});
----

=== Array Literals

[source,java]
----
Expr arrayLit = Expr.arrayLiteral(
    Expr.decimal(1), Expr.decimal(2), Expr.decimal(3)
);  // { 1, 2, 3 }
----

=== Unqualified Method Calls

For calls without a receiver (e.g., inherited or imported static methods):

[source,java]
----
Expr call = Expr.callPlain("println", Expr.str("hello"));  // println("hello")

=== Complete Expressions Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.Var;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class ExpressionsExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        sources.createSourceFile("com.example", "MathUtils", sf -> {
            sf.class_("MathUtils", cc -> {
                cc.public_();
                cc.final_();

                // public static int clamp(int value, int min, int max)
                cc.method("clamp", mc -> {
                    mc.public_();
                    mc.static_();
                    mc.returning(Type.INT);
                    mc.param("value", Type.INT);
                    mc.param("min", Type.INT);
                    mc.param("max", Type.INT);
                    mc.body(body -> {
                        // return value < min ? min : value > max ? max : value;
                        Var value = Expr.$v("value");
                        Var min = Expr.$v("min");
                        Var max = Expr.$v("max");
                        body.return_(
                            value.lt(min).cond(min,
                                value.gt(max).cond(max, value))
                        );
                    });
                });

                // public static boolean isPowerOfTwo(int n)
                cc.method("isPowerOfTwo", mc -> {
                    mc.public_();
                    mc.static_();
                    mc.returning(Type.BOOLEAN);
                    mc.param("n", Type.INT);
                    mc.body(body -> {
                        // return n > 0 && (n & (n - 1)) == 0;
                        Var n = Expr.$v("n");
                        body.return_(
                            n.gt(Expr.ZERO).and(
                                n.bitAnd(n.sub(Expr.ONE).paren())
                                    .eq(Expr.ZERO)
                            )
                        );
                    });
                });
            });
        });

        sources.writeSources();
    }
}

== Source Files and Imports

The `SourceFileCreator` defines the contents of a `.java` file: imports and top-level type declarations.

=== Import Declarations

[source,java]

sources.createSourceFile("com.example", "MyClass", sf -> {

import java.nio.file.Path;

import io.smallrye.jdeparser.JDeparser;

=== Complete Interface Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;
import java.util.List;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.creator.ModifierFlag;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class InterfaceExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        Type listType = Type.of(List.class);

        sources.createSourceFile("com.example", "Repository", sf -> {
            sf.import_(List.class);
            sf.import_(Type.named("java.util.Optional"));

            sf.interface_("Repository", ifc -> {
                ifc.public_();
                ifc.sealed_();
                ifc.typeParam("T", tp -> {});
                ifc.typeParam("ID", tp -> {});
                ifc.permits(Type.named("com.example.AbstractRepository"));

                // Optional<T> findById(ID id);
                ifc.method("findById", mc -> {
                    mc.returning(Type.named("java.util.Optional")
                        .typeArg(Type.named("T")));
                    mc.param("id", Type.named("ID"));
                });

                // List<T> findAll();
                ifc.method("findAll", mc -> {
                    mc.returning(listType.typeArg(Type.named("T")));
                });

                // T save(T entity);
                ifc.method("save", mc -> {
                    mc.returning(Type.named("T"));
                    mc.param("entity", Type.named("T"));
                });

                // default void deleteById(ID id) { ... }
                ifc.method("deleteById", mc -> {
                    mc.addFlag(ModifierFlag.DEFAULT);
                    mc.returning(Type.VOID);
                    mc.param("id", Type.named("ID"));
                    mc.body(body -> {
                        body.emit(Expr.callPlain("findById", Expr.$v("id"))
                            .call("ifPresent",
                                Expr.lambda("e",
                                    Expr.callPlain("delete", Expr.$v("e")))));
                    });
                });

                // void delete(T entity);
                ifc.method("delete", mc -> {
                    mc.returning(Type.VOID);
                    mc.param("entity", Type.named("T"));
                });

                // Nested listener interface
                ifc.interface_("ChangeListener", inner -> {
                    inner.typeParam("E", tp -> {});

                    inner.method("onSave", mc -> {
                        mc.returning(Type.VOID);
                        mc.param("entity", Type.named("E"));
                    });

                    inner.method("onDelete", mc -> {
                        mc.returning(Type.VOID);
                        mc.param("entity", Type.named("E"));
                    });
                });
            });
        });

        sources.writeSources();
    }
}

=== Basic Annotation

=== Annotation with Elements

[source,java]
----
sf.annotationInterface_("RestEndpoint", ai -> {
    ai.public_();

    // Add meta-annotations
    ai.annotate(Type.named("java.lang.annotation.Retention"), ann -> {
        ann.value(Type.named("java.lang.annotation.RetentionPolicy")
            .field("RUNTIME"));
    });
    ai.annotate(Type.named("java.lang.annotation.Target"), ann -> {
        ann.value(Type.named("java.lang.annotation.ElementType")
            .field("METHOD"));
    });

    // String path();
    ai.element("path", Type.STRING);

    // String method() default "GET";
    ai.element("method", Type.STRING, Expr.str("GET"));

    // String[] produces() default {};
    ai.element("produces", Type.STRING.array(),
        Expr.arrayLiteral());

    // int timeout() default 30;

sf.annotationInterface_("Severity", ai -> {

[[methods-constructors]]
== Methods and Constructors

The `MethodCreator` and `ConstructorCreator` interfaces configure method and constructor declarations.
Both support parameters, thrown exceptions, type parameters, and a body.
Methods additionally have a return type.

=== Method Signatures

[source,java]
----
cc.method("process", mc -> {
    mc.public_();
    mc.returning(Type.VOID);
    mc.param("input", Type.STRING);
    mc.param("count", Type.INT);
    mc.body(body -> {
        // ...
    });
});
----

=== Return Types

Every method needs a return type.
Use `Type.VOID` for void methods:

[source,java]
----
// void shutdown()
cc.method("shutdown", mc -> {
    mc.public_();
    mc.returning(Type.VOID);
    mc.body(body -> {});
});

// String getName()
cc.method("getName", mc -> {
    mc.public_();
    mc.returning(Type.STRING);
    mc.body(body -> {
        body.return_(Expr.$v("name"));
    });
});
----

=== Parameterized Methods

Parameters can be configured with annotations and documentation via the callback form:

[source,java]
----
cc.method("find", mc -> {
    mc.public_();
    mc.returning(Type.STRING);

    // Simple parameter
    mc.param("id", Type.LONG);

    // Configured parameter with annotation and Javadoc
    mc.param("name", Type.STRING, pc -> {
        pc.annotate(Type.named("jakarta.validation.constraints.NotNull"));
        pc.final_();
        pc.docComment(doc -> doc.text("the name to search for"));
    });

    mc.body(body -> {
        body.return_(Expr.NULL);
    });
});
----

=== Varargs Parameters

Use `varargParam` for variable-arity parameters.
The type argument is the _element_ type, not the array type:

[source,java]
----
cc.method("format", mc -> {
    mc.public_();
    mc.static_();
    mc.returning(Type.STRING);
    mc.param("pattern", Type.STRING);
    mc.varargParam("args", Type.OBJECT, pc -> {});
    mc.body(body -> {
        body.return_(Type.STRING.call("format",
            Expr.$v("pattern"), Expr.$v("args")));
    });
});
----

=== Thrown Exceptions

[source,java]
----
cc.method("readFile", mc -> {
    mc.public_();
    mc.returning(Type.STRING);
    mc.param("path", Type.STRING);
    mc.throws_(Type.named("java.io.IOException"));
    mc.throws_(Type.named("java.lang.SecurityException"));
    mc.body(body -> {
        body.return_(Expr.str(""));
    });
});
----

=== Generic Methods

[source,java]
----
cc.method("convert", mc -> {
    mc.public_();
    mc.typeParam("T", tp -> {});
    mc.typeParam("R", tp -> {
        tp.extends_(Type.named("java.io.Serializable"));
    });
    mc.returning(Type.named("R"));
    mc.param("input", Type.named("T"));
    mc.param("converter",
        Type.named("java.util.function.Function")
            .typeArg(Type.named("T"), Type.named("R")));
    mc.body(body -> {
        body.return_(Expr.$v("converter").call("apply",
            Expr.$v("input")));
    });
});
----

=== Abstract and Native Methods

Abstract and native methods have no body:

[source,java]
----
// abstract int compute(int x);
cc.method("compute", mc -> {
    mc.abstract_();
    mc.returning(Type.INT);
    mc.param("x", Type.INT);
});

// native void init();
cc.method("init", mc -> {
    mc.native_();
    mc.returning(Type.VOID);
});
----

=== Constructors

Constructors are similar to methods but without a return type:

[source,java]
----
cc.constructor(con -> {
    con.public_();
    con.param("name", Type.STRING);
    con.param("age", Type.INT);
    con.body(body -> {
        body.emit(Expr.THIS.field("name").assign(Expr.$v("name")));
        body.emit(Expr.THIS.field("age").assign(Expr.$v("age")));
    });
});
----

=== Constructor Delegation

Use `callThis` or `callSuper` in the body to delegate to another constructor:

[source,java]
----
// Default constructor delegates to parameterized constructor
cc.constructor(con -> {
    con.public_();
    con.body(body -> {
        body.callThis(Expr.str("unnamed"), Expr.ZERO);
    });
});

// Parameterized constructor delegates to super
cc.constructor(con -> {
    con.public_();
    con.param("name", Type.STRING);
    con.param("id", Type.INT);
    con.body(body -> {
        body.callSuper(Expr.$v("name"));
        body.emit(Expr.THIS.field("id").assign(Expr.$v("id")));
    });
});
----

=== Complete Methods and Constructors Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class MethodsExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        sources.createSourceFile("com.example", "HttpClient", sf -> {
            sf.import_(Type.named("java.util.Map"));
            sf.import_(Type.named("java.util.List"));
            sf.import_(Type.named("java.io.IOException"));

            sf.class_("HttpClient", cc -> {
                cc.public_();

                cc.field("baseUrl", fc -> {
                    fc.private_();
                    fc.final_();
                    fc.type(Type.STRING);
                });

                cc.field("timeout", fc -> {
                    fc.private_();
                    fc.type(Type.INT);
                });

                // Primary constructor
                cc.constructor(con -> {
                    con.public_();
                    con.param("baseUrl", Type.STRING);
                    con.param("timeout", Type.INT);
                    con.body(body -> {
                        body.emit(Expr.THIS.field("baseUrl")
                            .assign(Expr.$v("baseUrl")));
                        body.emit(Expr.THIS.field("timeout")
                            .assign(Expr.$v("timeout")));
                    });
                });

                // Delegating constructor
                cc.constructor(con -> {
                    con.public_();
                    con.param("baseUrl", Type.STRING);
                    con.body(body -> {
                        body.callThis(Expr.$v("baseUrl"), Expr.decimal(30000));
                    });
                });

                // Generic method with exception
                cc.method("get", mc -> {
                    mc.public_();
                    mc.typeParam("T", tp -> {});
                    mc.returning(Type.named("T"));
                    mc.param("path", Type.STRING);
                    mc.param("responseType", Type.named("java.lang.Class")
                        .typeArg(Type.named("T")));
                    mc.throws_(Type.named("java.io.IOException"));
                    mc.body(body -> {
                        body.throw_(Type.named("java.lang.UnsupportedOperationException")
                            .new_(Expr.str("not implemented")));
                    });
                });

                // Varargs method
                cc.method("buildUrl", mc -> {
                    mc.private_();
                    mc.returning(Type.STRING);
                    mc.param("base", Type.STRING);
                    mc.varargParam("segments", Type.STRING, pc -> {});
                    mc.body(body -> {
                        body.return_(Expr.$v("base"));
                    });
                });
            });
        });

        sources.writeSources();
    }
}

[[modifiers-annotations]]
== Modifiers and Annotations

=== The Modifier System

Modifiers are controlled through `ModifiableCreator`, which provides:

* **`setAccess(AccessLevel)`** — sets the access level
* **`addFlag(ModifierFlag)`** — adds a non-access modifier
* **`removeFlag(ModifierFlag)`** — removes a non-access modifier

==== Access Levels

The `AccessLevel` enum has four constants:

[cols="1,2",options="header"]
|===
|Constant |Keyword

|`PUBLIC` |`public`
|`PROTECTED` |`protected`
|`PACKAGE_PRIVATE` |_(no keyword)_
|`PRIVATE` |`private`
|===

Convenience methods on `ModifiableCreator` map directly:

[source,java]
----
mc.public_();         // mc.setAccess(AccessLevel.PUBLIC)
mc.protected_();      // mc.setAccess(AccessLevel.PROTECTED)
mc.packagePrivate();  // mc.setAccess(AccessLevel.PACKAGE_PRIVATE)
mc.private_();        // mc.setAccess(AccessLevel.PRIVATE)
----

==== Modifier Flags

The `ModifierFlag` enum provides non-access modifiers:

[cols="1,2",options="header"]
|===
|Constant |Keyword

|`ABSTRACT` |`abstract`
|`FINAL` |`final`
|`STATIC` |`static`
|`SYNCHRONIZED` |`synchronized`
|`NATIVE` |`native`
|`STRICTFP` |`strictfp`
|`TRANSIENT` |`transient`
|`VOLATILE` |`volatile`
|`DEFAULT` |`default`
|`SEALED` |`sealed`
|`NON_SEALED` |`non-sealed`
|`VARARGS` |`...` _(internal use)_
|===

Convenience methods:

[source,java]
----
mc.final_();        // mc.addFlag(ModifierFlag.FINAL)
mc.static_();       // mc.addFlag(ModifierFlag.STATIC)
mc.abstract_();     // mc.addFlag(ModifierFlag.ABSTRACT)
mc.synchronized_(); // mc.addFlag(ModifierFlag.SYNCHRONIZED)
mc.native_();       // mc.addFlag(ModifierFlag.NATIVE)
mc.strictfp_();     // mc.addFlag(ModifierFlag.STRICTFP)
mc.sealed_();       // mc.addFlag(ModifierFlag.SEALED)
mc.nonSealed_();    // mc.addFlag(ModifierFlag.NON_SEALED)
----

==== Modifier Locations

Each declaration type has a `ModifierLocation` that constrains which modifiers are valid.
Invalid modifiers are rejected at call time with `IllegalArgumentException`.

[cols="1,3,3",options="header"]
|===
|Location |Valid Access Levels |Valid Flags

|`CLASS` |all |`abstract`, `final`, `static`, `strictfp`, `sealed`, `non-sealed`
|`INTERFACE` |all |`abstract`, `static`, `strictfp`, `sealed`, `non-sealed`
|`ENUM` |all |`static`, `strictfp`
|`RECORD` |all |`static`, `final`, `sealed`, `non-sealed`
|`ANNOTATION` |all |`abstract`, `static`
|`METHOD` |all |`abstract`, `final`, `static`, `synchronized`, `native`, `strictfp`, `default`
|`CONSTRUCTOR` |all |_(none)_
|`FIELD` |all |`final`, `static`, `transient`, `volatile`
|`INTERFACE_METHOD` |`public`, `private`, `package-private` |`abstract`, `static`, `default`, `strictfp`
|`INTERFACE_FIELD` |`public`, `package-private` |`final`, `static`
|`PARAMETER` |_(none)_ |`final`, `varargs`
|`LOCAL_VARIABLE` |_(none)_ |`final`
|===

==== Mutual Exclusion

Certain modifier flags are mutually exclusive and cannot be combined:

* `abstract` and `final`
* `abstract` and `native`
* `final` and `volatile`
* `final` and `sealed`
* `final` and `non-sealed`
* `sealed` and `non-sealed`

Attempting to add a flag that conflicts with an existing flag throws `IllegalArgumentException`.

=== Annotations

==== Adding Annotations

The `AnnotatableCreator` interface (parent of `ModifiableCreator`) provides annotation support.
It is also implemented by `EnumConstantCreator`, `LocalVarCreator`, and `RecordComponentCreator`.

[source,java]
----
// Marker annotation (no members)
mc.annotate(Type.named("java.lang.Override"));

// Annotation with default value member
mc.annotate(Type.named("java.lang.SuppressWarnings"), ann -> {
    ann.value(Expr.str("unchecked"));
});

// Annotation with named members
mc.annotate(Type.named("jakarta.persistence.Column"), ann -> {
    ann.member("name", Expr.str("user_name"));
    ann.member("nullable", Expr.FALSE);
    ann.member("length", Expr.decimal(255));
});

// Annotation with array member
mc.annotate(Type.named("jakarta.persistence.NamedQueries"), ann -> {
    ann.memberArray("value",
        // Each element would be a nested annotation expression
        Expr.str("query1"),
        Expr.str("query2")
    );
});
----

==== Annotating Parameters

[source,java]
----
mc.param("name", Type.STRING, pc -> {
    pc.annotate(Type.named("jakarta.validation.constraints.NotNull"));
    pc.annotate(Type.named("jakarta.validation.constraints.Size"), ann -> {
        ann.member("min", Expr.decimal(1));
        ann.member("max", Expr.decimal(100));
    });
});
----

==== Annotating Local Variables

[source,java]
----
body.var(Type.STRING, "result", Expr.str(""), lvc -> {
    lvc.annotate(Type.named("org.checkerframework.checker.nullness.qual.NonNull"));
});
----


[[control-flow]]
== Control Flow

The `BlockCreator` is the central interface for building method bodies and other statement blocks.
It provides methods for all Java control flow constructs.

=== Expression Statements

Use `emit` to add an expression as a statement:

[source,java]
----
body.emit(Type.named("java.lang.System").field("out")
    .call("println", Expr.str("Hello")));

body.emit(Expr.$v("count").postInc());

body.emit(Expr.$v("list").call("add", Expr.$v("item")));
----

=== Local Variables

==== Typed Variables

[source,java]
----
Var count = body.var(Type.INT, "count", Expr.ZERO);
Var name = body.var(Type.STRING, "name", Expr.str("default"));
----

The returned `Var` can be used in subsequent expressions:

[source,java]
----
body.emit(count.assign(count.add(Expr.ONE)));
body.emit(name.call("toUpperCase"));
----

==== Type-Inferred Variables (Java 10+)

[source,java]
----
Var items = body.var("items",
    Type.named("java.util.ArrayList").new_());
----

==== Annotated and Final Variables

[source,java]
----
Var value = body.var(Type.STRING, "value", Expr.str(""), lvc -> {
    lvc.final_();
    lvc.annotate(Type.named("org.example.NonNull"));
});
----

=== Conditionals

==== Simple If

[source,java]
----
body.if_(Expr.$v("x").gt(Expr.ZERO), then -> {
    then.emit(Expr.$v("result").assign(Expr.str("positive")));
});
----

==== If-Else

[source,java]
----
body.ifElse(Expr.$v("x").gt(Expr.ZERO),
    then -> {
        then.emit(Expr.$v("result").assign(Expr.str("positive")));
    },
    else_ -> {
        else_.emit(Expr.$v("result").assign(Expr.str("non-positive")));
    }
);
----

==== Chained If-Else

Chain `if_` and `ifElse` calls within the else block:

[source,java]
----
body.ifElse(Expr.$v("x").gt(Expr.ZERO),
    then -> {
        then.return_(Expr.str("positive"));
    },
    else1 -> {
        else1.ifElse(Expr.$v("x").lt(Expr.ZERO),
            then -> {
                then.return_(Expr.str("negative"));
            },
            else2 -> {
                else2.return_(Expr.str("zero"));
            }
        );
    }
);
----

=== Loops

==== While Loop

[source,java]
----
body.while_(Expr.$v("iterator").call("hasNext"), loop -> {
    Var item = loop.var("item",
        Expr.$v("iterator").call("next"));
    loop.emit(Expr.$v("result").call("add", item));
});
----

==== Do-While Loop

Note that `doWhile` takes the body first, then the condition:

[source,java]
----
body.doWhile(
    loop -> {
        loop.emit(Expr.$v("line").assign(
            Expr.$v("reader").call("readLine")));
    },
    Expr.$v("line").ne(Expr.NULL)
);
----

==== Traditional For Loop

The `ForCreator` provides `init`, `condition`, `update`, and `body`:

[source,java]
----
body.for_(fc -> {
    Var i = fc.init(Type.INT, "i", Expr.ZERO);
    fc.condition(i.lt(Expr.$v("array").field("length")));
    fc.update(i.postInc());
    fc.body(loop -> {
        loop.emit(Type.named("java.lang.System").field("out")
            .call("println", Expr.$v("array").idx(i)));
    });
});
----

==== Enhanced For-Each Loop

[source,java]
----
Var item = body.forEach(Type.STRING, "item", Expr.$v("items"),
    loop -> {
        loop.emit(Type.named("java.lang.System").field("out")
            .call("println", item));
    }
);
----

NOTE: The `forEach` method returns the loop variable so it can be used within the body.
The variable is also accessible inside the callback.

=== Switch Statements

==== Constant Cases

[source,java]
----
body.switch_(Expr.$v("direction"), sw -> {
    sw.case_(Expr.$v("NORTH"), cb -> {
        cb.emit(Expr.$v("y").postInc());
        cb.break_();
    });
    sw.case_(Expr.$v("SOUTH"), cb -> {
        cb.emit(Expr.$v("y").postDec());
        cb.break_();
    });
    sw.default_(cb -> {
        cb.throw_(Type.named("java.lang.IllegalArgumentException")
            .new_(Expr.str("Unknown direction")));
    });
});
----

==== Multi-Value Cases

[source,java]
----
body.switch_(Expr.$v("dayOfWeek"), sw -> {
    sw.case_(java.util.List.of(Expr.decimal(6), Expr.decimal(7)), cb -> {
        cb.return_(Expr.TRUE);
    });
    sw.default_(cb -> {
        cb.return_(Expr.FALSE);
    });
});
----

==== Type Pattern Cases (Java 21+)

[source,java]
----
body.switch_(Expr.$v("shape"), sw -> {
    sw.case_(Type.named("com.example.Circle"), "c", cc -> {
        cc.body(cb -> {
            cb.return_(Expr.$v("c").call("radius").mul(
                Expr.$v("c").call("radius")).mul(Expr.$v("Math").field("PI")));
        });
    });
    sw.case_(Type.named("com.example.Rectangle"), "r", cc -> {
        cc.when_(Expr.$v("r").call("width").gt(Expr.ZERO));
        cc.body(cb -> {
            cb.return_(Expr.$v("r").call("width").mul(
                Expr.$v("r").call("height")));
        });
    });
    sw.caseNull(cb -> {
        cb.return_(Expr.ZERO);
    });
    sw.default_(cb -> {
        cb.throw_(Type.named("java.lang.IllegalArgumentException")
            .new_(Expr.str("Unknown shape")));
    });
});
----

TIP: Type pattern cases use `CaseCreator`, which provides `when_(Expr)` for guard conditions
and `body(Consumer<BlockCreator>)` for the case body.

=== Exception Handling

==== Basic Try-Catch

[source,java]
----
body.try_(tc -> {
    tc.body(tb -> {
        tb.emit(Expr.$v("connection").call("execute"));
    });
    tc.catch_(Type.named("java.sql.SQLException"), "e", cb -> {
        cb.emit(Expr.$v("logger").call("error",
            Expr.str("Query failed"), Expr.$v("e")));
    });
});
----

==== Multi-Catch

[source,java]
----
body.try_(tc -> {
    tc.body(tb -> {
        tb.emit(Expr.$v("parser").call("parse", Expr.$v("input")));
    });
    tc.catch_(
        java.util.List.of(
            Type.named("java.io.IOException"),
            Type.named("java.text.ParseException")
        ),
        "e",
        cb -> {
            cb.emit(Expr.$v("logger").call("warn",
                Expr.$v("e").call("getMessage")));
        }
    );
});
----

==== Try-with-Resources

[source,java]
----
body.try_(tc -> {
    tc.with(Type.named("java.io.BufferedReader"), "reader",
        Type.named("java.io.BufferedReader").new_(
            Type.named("java.io.FileReader").new_(Expr.$v("path"))));
    tc.body(tb -> {
        Var line = tb.var(Type.STRING, "line",
            Expr.$v("reader").call("readLine"));
        tb.while_(line.ne(Expr.NULL), loop -> {
            loop.emit(Expr.$v("result").call("add", line));
            loop.emit(line.assign(Expr.$v("reader").call("readLine")));
        });
    });
});
----

==== Try-Catch-Finally

[source,java]
----
body.try_(tc -> {
    tc.body(tb -> {
        tb.emit(Expr.$v("lock").call("lock"));
        tb.emit(Expr.$v("resource").call("use"));

    tc.catch_(Type.named("java.lang.Exception"), "e", cb -> {
        cb.emit(Expr.$v("logger").call("error",
            Expr.str("Error"), Expr.$v("e")));

=== Labels

Use `labeled` to create a labeled statement, then reference the label with `break_` or `continue_`:

[source,java]
----
body.labeled("outer", (outerLabel, outerBody) -> {
    outerBody.for_(fc -> {
        Var i = fc.init(Type.INT, "i", Expr.ZERO);
        fc.condition(i.lt(Expr.$v("rows")));
        fc.update(i.postInc());
        fc.body(outerLoop -> {
            outerLoop.for_(fc2 -> {
                Var j = fc2.init(Type.INT, "j", Expr.ZERO);
                fc2.condition(j.lt(Expr.$v("cols")));
                fc2.update(j.postInc());
                fc2.body(innerLoop -> {
                    innerLoop.if_(Expr.$v("matrix").idx(i).idx(j)
                            .eq(Expr.$v("target")),
                        found -> {
                            found.break_(outerLabel);
                        }
                    );
                });
            });
        });
    });
});
----

[source,java]
----
// return (void)
body.return_();

// return expression
body.return_(Expr.$v("result"));

// throw
body.throw_(Type.named("java.lang.IllegalStateException")

== Javadoc Comments

JDeparser 3 provides comprehensive support for generating Javadoc comments.
Documentation is set via the `docComment` callback on any `DocCommentableCreator`.

=== Basic Documentation

[source,java]

cc.method("getName", mc -> {
    mc.public_();
    mc.returning(Type.STRING);
    mc.docComment(doc -> {
        doc.text("Returns the name of this entity.");

    mc.body(body -> {
        body.return_(Expr.$v("name"));

The `DocInlineCreator` provides inline Javadoc tags that can appear within text:

mc.docComment(doc -> {

=== Inline Return Tag (Java 16+)

The `returnInline` method generates `{@return ...}`, which serves as both the summary sentence and the `@return` tag:

[source,java]
----

    doc.returnInline("the current count of processed items");
});
----

This generates:

[source,java]
----

 * {@return the current count of processed items}
 */
----

=== DocReference

The `DocReference` interface represents a `Type#member` reference for use in `@see`, `{@link}`, and `{@value}` tags.
Create references via `Type.docRef(String)` or `DocReference.of(String)`:

[source,java]
----
// @see String#length()
doc.see(Type.STRING.docRef("length()"));

// {@link Map#of(Object, Object)}
doc.link(Type.named("java.util.Map").docRef("of(Object, Object)"));

// {@value Integer#MAX_VALUE}
doc.value(Type.named("java.lang.Integer").docRef("MAX_VALUE"));

// Rootless reference: {@link #myMethod()}
doc.link(DocReference.of("myMethod()"));
----

=== Automatic @param Generation

Parameters, type parameters, and record components automatically contribute `@param` tags when documented via their own `docComment` callback.
You do not add `@param` tags to the `DocCommentCreator` directly.

[source,java]
----
cc.method("merge", mc -> {
    mc.public_();
    mc.typeParam("T", tp -> {
        tp.extends_(Type.named("java.lang.Comparable").typeArg(Type.named("T")));
        tp.docComment(doc -> doc.text("the element type"));
    });
    mc.returning(Type.named("java.util.List").typeArg(Type.named("T")));

    mc.param("first", Type.named("java.util.List").typeArg(Type.named("T")), pc -> {
        pc.docComment(doc -> doc.text("the first list to merge"));
    });
    mc.param("second", Type.named("java.util.List").typeArg(Type.named("T")), pc -> {
        pc.docComment(doc -> doc.text("the second list to merge"));
    });

    mc.docComment(doc -> {
        doc.text("Merges two sorted lists into a single sorted list.");
        doc.return_("a new list containing all elements from both lists, sorted");
    });

    mc.body(body -> {
        body.throw_(Type.named("java.lang.UnsupportedOperationException").new_());
    });
});
----

This generates `@param <T>`, `@param first`, and `@param second` tags automatically from the individual `docComment` callbacks.

=== Complete Javadoc Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;
import java.util.List;

import io.smallrye.jdeparser.DocReference;
import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class JavadocExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        Type listType = Type.of(List.class);

        sources.createSourceFile("com.example", "DataProcessor", sf -> {
            sf.import_(List.class);

            sf.class_("DataProcessor", cc -> {
                cc.public_();

                cc.docComment(doc -> {
                    doc.text("Processes data records from various sources.");
                    doc.text("\n\n");
                    doc.text("This class provides methods for filtering, ");
                    doc.text("transforming, and aggregating data records. ");
                    doc.text("Use ");
                    doc.link(DocReference.of("process(List)"));
                    doc.text(" as the main entry point.");
                    doc.author("Code Generator");
                    doc.since("1.0");
                    doc.see(Type.named("com.example.DataRecord"));
                });

                cc.field("MAX_BATCH_SIZE", fc -> {
                    fc.public_();
                    fc.static_();
                    fc.final_();
                    fc.type(Type.INT);
                    fc.init(Expr.decimal(1000));
                    fc.docComment(doc -> {
                        doc.text("Maximum number of records to process in a single batch.");
                    });
                });

                cc.method("process", mc -> {
                    mc.public_();
                    mc.typeParam("T", tp -> {
                        tp.docComment(doc -> doc.text("the record type"));
                    });
                    mc.returning(listType.typeArg(Type.named("T")));
                    mc.param("records", listType.typeArg(Type.named("T")), pc -> {
                        pc.docComment(doc -> {
                            doc.text("the records to process (must not be ");
                            doc.code("null");
                            doc.text(")");
                        });
                    });
                    mc.throws_(Type.named("java.lang.IllegalArgumentException"));

                    mc.docComment(doc -> {
                        doc.text("Processes a list of records and returns the results.");
                        doc.return_(c -> {
                            c.text("a new list containing the processed records, ");
                            c.text("never ");
                            c.code("null");
                        });
                        doc.throws_(Type.named("java.lang.IllegalArgumentException"),
                            "if records is null or exceeds the maximum batch size");
                        doc.since("1.0");
                    });

                    mc.body(body -> {
                        body.return_(Expr.$v("records"));
                    });
                });
            });
        });

        sources.writeSources();
    }
}

[[source-version-features]]
== Source Version and Language Features

=== SourceVersion

The `SourceVersion` enum represents Java SE release versions from `JAVA_8` through `JAVA_26`.
It controls which language features are permitted during source generation.

[source,java]
----
Sources sources = JDeparser.createSources(
    filer, prefs, SourceVersion.JAVA_17
);
----

Key methods:

* **`version()`** — returns the numeric version (e.g., `17`)
* **`supports(LanguageFeature)`** — tests whether a feature is available
* **`require(LanguageFeature)`** — throws `IllegalStateException` if the feature is unavailable

[source,java]
----
SourceVersion sv = SourceVersion.JAVA_17;
sv.supports(LanguageFeature.RECORDS);             // true (Java 16+)
sv.supports(LanguageFeature.PATTERN_MATCHING_SWITCH); // false (Java 21+)

sv.require(LanguageFeature.SEALED_CLASSES);        // OK
sv.require(LanguageFeature.PATTERN_MATCHING_SWITCH); // throws IllegalStateException
----

=== LanguageFeature

Each `LanguageFeature` constant identifies a feature and its minimum `SourceVersion`.
The library checks these automatically when you use a version-gated API.

.Language Features by Version

|===

=== Version-Conditional Generation

You can query the source version to conditionally use features:

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.LanguageFeature;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class VersionExample {
    public static void main(String[] args) throws IOException {
        SourceVersion version = SourceVersion.JAVA_17;
        Filer filer = Filer.newInstance(Path.of("generated-sources"));
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), version
        );

        sources.createSourceFile("com.example", "Config", sf -> {
            sf.class_("Config", cc -> {
                cc.public_();

                cc.field("name", fc -> {
                    fc.private_();
                    fc.final_();
                    fc.type(Type.STRING);
                });

                cc.field("value", fc -> {
                    fc.private_();
                    fc.final_();
                    fc.type(Type.STRING);
                });

                cc.constructor(con -> {
                    con.public_();
                    con.param("name", Type.STRING);
                    con.param("value", Type.STRING);
                    con.body(body -> {
                        body.emit(Expr.THIS.field("name")
                            .assign(Expr.$v("name")));
                        body.emit(Expr.THIS.field("value")
                            .assign(Expr.$v("value")));
                    });
                });

                cc.method("describe", mc -> {
                    mc.public_();
                    mc.returning(Type.STRING);
                    mc.body(body -> {
                        if (version.supports(LanguageFeature.TEXT_BLOCKS)) {
                            // Use text blocks for Java 15+
                            body.return_(Expr.str("Config:\n  name=%s\n  value=%s")
                                .call("formatted",
                                    Expr.$v("name"), Expr.$v("value")));
                        } else {
                            // Fall back to concatenation for older versions
                            body.return_(
                                Expr.str("Config: name=")
                                    .add(Expr.$v("name"))
                                    .add(Expr.str(", value="))
                                    .add(Expr.$v("value")));
                        }
                    });
                });
            });
        });

        sources.writeSources();
    }
}
----

NOTE: Version-gated validation happens at construction time.
For example, calling `sealed_()` when the target is `JAVA_8` will throw immediately.
Use `SourceVersion.supports()` in your generator to choose the right API calls for the target version.


[[formatting]]
== Formatting Configuration

The `FormatPreferences` class controls all aspects of source code formatting.
Use `FormatPreferences.defaults()` for sensible defaults, or `FormatPreferences.builder()` to customize.

=== Indentation

The `Indentation` enum defines six contexts with default sizes:

[cols="1,1,2",options="header"]
|===
|Context |Default |Description

|`MEMBERS_TOP_LEVEL` |4 |Class/interface body members
|`LABELS` |0 |Labels
|`CASE_LABELS` |0 |Case labels within switch
|`LINE_CONTINUATION` |4 |Continuation of wrapped lines
|`LINE` |4 |Block-level content
|`HTML_TAG` |2 |HTML tags in Javadoc
|===

[source,java]
----
FormatPreferences prefs = FormatPreferences.builder()
    .indent(FormatPreferences.Indentation.MEMBERS_TOP_LEVEL, 2)
    .indent(FormatPreferences.Indentation.LINE, 2)
    .indent(FormatPreferences.Indentation.LINE_CONTINUATION, 4)
    .build();
----

Use `indentAbsolute` for absolute positioning (replaces parent indent instead of adding to it):

[source,java]
----
FormatPreferences prefs = FormatPreferences.builder()
    .indent(FormatPreferences.Indentation.LABELS, 0)
    .indentAbsolute(FormatPreferences.Indentation.LABELS)
    .build();
----

=== Spacing

The `Space` enum defines spacing contexts — over 90 specific syntactic positions.
Each can be set to `NONE`, `SPACE`, or `NEWLINE` via `SpaceType`.

Categories of spacing contexts include:

* **Before parentheses** — e.g., `BEFORE_PAREN_IF`, `BEFORE_PAREN_METHOD_CALL`
* **Within parentheses** — e.g., `WITHIN_PAREN_IF`, `WITHIN_PAREN_METHOD_CALL_EMPTY`
* **Before braces** — e.g., `BEFORE_BRACE_CLASS`, `BEFORE_BRACE_METHOD`, `BEFORE_BRACE_IF`
* **Within braces** — e.g., `WITHIN_BRACES_CODE`, `WITHIN_BRACES_EMPTY`
* **Around operators** — e.g., `AROUND_ASSIGN`, `AROUND_LOGICAL`, `AROUND_ARROW`
* **Before keywords** — e.g., `BEFORE_KEYWORD_ELSE`, `BEFORE_KEYWORD_CATCH`
* **Commas** — e.g., `AFTER_COMMA`, `COMMA_ENUM_CONSTANT`
* **Ternary** — `BEFORE_TERNARY_Q`, `AFTER_TERNARY_Q`, `BEFORE_TERNARY_COLON`, `AFTER_TERNARY_COLON`
* **Annotations** — `AFTER_ANNOTATION`, `AFTER_PARAM_ANNOTATION`
* **Declarations** — `BEFORE_METHOD`, `BEFORE_CLASS`

Example — Allman brace style (opening brace on new line):

[source,java]
----
FormatPreferences.Builder b = FormatPreferences.builder();
// Set all BEFORE_BRACE_* to NEWLINE for Allman style
b.space(FormatPreferences.Space.BEFORE_BRACE_CLASS, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_INTERFACE, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_METHOD, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_IF, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_ELSE, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_FOR, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_WHILE, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_DO, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_SWITCH, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_TRY, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_CATCH, FormatPreferences.SpaceType.NEWLINE);
b.space(FormatPreferences.Space.BEFORE_BRACE_FINALLY, FormatPreferences.SpaceType.NEWLINE);
FormatPreferences allman = b.build();
----

=== Wrapping

The `Wrapping` enum defines list contexts that can be wrapped:

[cols="1,1,2",options="header"]
|===
|Context |Default |Description

|`EXCEPTION_LIST` |`WRAP_ONLY_IF_LONG` |`throws` clause
|`IMPLEMENTS_LIST` |`WRAP_ONLY_IF_LONG` |`implements` clause
|`EXTENDS_LIST` |`WRAP_ONLY_IF_LONG` |`extends` clause
|`PERMITS_LIST` |`WRAP_ONLY_IF_LONG` |`permits` clause
|`PARAMETER_LIST` |`WRAP_ONLY_IF_LONG` |Method parameter declarations
|`ARGUMENT_LIST` |`WRAP_ONLY_IF_LONG` |Method call arguments

        FormatPreferences.WrappingMode.WRAP_ONLY_IF_LONG)

props.setProperty("space.before-brace-class", "NEWLINE");
props.setProperty("wrapping.parameter-list", "ALWAYS_WRAP");
props.setProperty("option.enum-trailing-comma", "true");

FormatPreferences prefs = FormatPreferences.builder()
    .fromProperties(props)
    .build();
----

Property key naming convention: lowercase, hyphen-separated form of the enum constant name (e.g., `MEMBERS_TOP_LEVEL` becomes `members-top-level`), prefixed with the category (`indent.`, `space.`, `wrapping.`, `option.`).

=== Complete Formatting Example

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class FormattingExample {
    public static void main(String[] args) throws IOException {
        Filer filer = Filer.newInstance(Path.of("generated-sources"));

        // Custom formatting: 2-space indent, 80 columns, Allman braces
        FormatPreferences prefs = FormatPreferences.builder()
            .indent(FormatPreferences.Indentation.MEMBERS_TOP_LEVEL, 2)
            .indent(FormatPreferences.Indentation.LINE, 2)
            .lineLength(80)
            .space(FormatPreferences.Space.BEFORE_BRACE_CLASS,
                FormatPreferences.SpaceType.NEWLINE)
            .space(FormatPreferences.Space.BEFORE_BRACE_METHOD,
                FormatPreferences.SpaceType.NEWLINE)
            .addOption(FormatPreferences.Opt.ENUM_TRAILING_COMMA)
            .wrapMode(FormatPreferences.Wrapping.PARAMETER_LIST,
                FormatPreferences.WrappingMode.ALWAYS_WRAP)
            .build();

        Sources sources = JDeparser.createSources(
            filer, prefs, SourceVersion.JAVA_17
        );

        sources.createSourceFile("com.example", "StyledClass", sf -> {
            sf.class_("StyledClass", cc -> {
                cc.public_();

                cc.field("value", fc -> {
                    fc.private_();
                    fc.type(Type.INT);
                });

                cc.constructor(con -> {
                    con.public_();
                    con.param("value", Type.INT);
                    con.body(body -> {
                        body.emit(Expr.THIS.field("value")
                            .assign(Expr.$v("value")));
                    });
                });

                cc.method("getValue", mc -> {
                    mc.public_();
                    mc.returning(Type.INT);
                    mc.body(body -> {
                        body.return_(Expr.$v("value"));
                    });
                });
            });
        });

        sources.writeSources();
    }
}

[[use-case-annotation-processor]]
== Use Case: Annotation Processor

A common use of JDeparser 3 is within annotation processors.
The `Filer` class integrates with `javax.annotation.processing.Filer` for seamless source file creation.

=== Complete Annotation Processor Example

This example defines a `@GenerateBuilder` annotation and the processor that generates builder classes:

[source,java]
----
import java.io.IOException;
import java.util.Set;
import java.util.List;
import java.util.ArrayList;

import javax.annotation.processing.AbstractProcessor;
import javax.annotation.processing.RoundEnvironment;
import javax.annotation.processing.SupportedAnnotationTypes;
import javax.annotation.processing.SupportedSourceVersion;
import javax.lang.model.element.Element;
import javax.lang.model.element.ElementKind;
import javax.lang.model.element.TypeElement;
import javax.lang.model.element.VariableElement;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

@SupportedAnnotationTypes("com.example.GenerateBuilder")
@SupportedSourceVersion(javax.lang.model.SourceVersion.RELEASE_17)
public class BuilderProcessor extends AbstractProcessor {

    @Override
    public boolean process(Set<? extends TypeElement> annotations,
                          RoundEnvironment roundEnv) {
        for (Element element : roundEnv.getElementsAnnotatedWith(
                annotations.iterator().next())) {
            if (element.getKind() != ElementKind.CLASS) {
                continue;
            }
            TypeElement typeElement = (TypeElement) element;
            try {
                generateBuilder(typeElement);
            } catch (IOException e) {
                throw new RuntimeException("Failed to generate builder", e);
            }
        }
        return true;
    }

    private void generateBuilder(TypeElement typeElement) throws IOException {
        String packageName = processingEnv.getElementUtils()
            .getPackageOf(typeElement).getQualifiedName().toString();
        String className = typeElement.getSimpleName().toString();
        String builderName = className + "Builder";

        // Collect fields
        List<VariableElement> fields = new ArrayList<>();
        for (Element enclosed : typeElement.getEnclosedElements()) {
            if (enclosed.getKind() == ElementKind.FIELD) {
                fields.add((VariableElement) enclosed);
            }
        }

        // Use JDeparser with the annotation processing Filer
        Filer filer = Filer.newInstance(processingEnv.getFiler());
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        Type targetType = Type.named(packageName + "." + className);

        sources.createSourceFile(packageName, builderName, sf -> {
            sf.class_(builderName, cc -> {
                cc.public_();
                cc.final_();

                // Generate a field for each target field
                for (VariableElement field : fields) {
                    String fieldName = field.getSimpleName().toString();
                    Type fieldType = Type.of(field.asType());

                    cc.field(fieldName, fc -> {
                        fc.private_();
                        fc.type(fieldType);
                    });
                }

                // Generate a setter for each field (returns this)
                Type builderType = Type.named(packageName + "." + builderName);
                for (VariableElement field : fields) {
                    String fieldName = field.getSimpleName().toString();
                    Type fieldType = Type.of(field.asType());

                    cc.method(fieldName, mc -> {
                        mc.public_();
                        mc.returning(builderType);
                        mc.param(fieldName, fieldType);
                        mc.body(body -> {
                            body.emit(Expr.THIS.field(fieldName)
                                .assign(Expr.$v(fieldName)));
                            body.return_(Expr.THIS);
                        });
                    });
                }

                // Generate build() method
                cc.method("build", mc -> {
                    mc.public_();
                    mc.returning(targetType);
                    mc.body(body -> {
                        // new TargetClass(field1, field2, ...)
                        Expr[] args = fields.stream()
                            .map(f -> Expr.$v(f.getSimpleName().toString()))
                            .toArray(Expr[]::new);
                        body.return_(targetType.new_(args));
                    });
                });
            });
        });

        sources.writeSources();
    }
}
----

Key points for annotation processor integration:

* Use `Filer.newInstance(processingEnv.getFiler())` to integrate with the annotation processing environment
* Use `Type.of(TypeMirror)` to convert `javax.lang.model` types to JDeparser types
* Generated sources are automatically registered with the compilation environment


[[use-case-build-time]]
== Use Case: Build-Time Code Generator

JDeparser 3 is also useful for standalone code generators invoked during the build.
This example generates data classes from a simple definition.

=== Complete Build-Time Generator

[source,java]
----
import java.io.IOException;
import java.nio.file.Path;
import java.util.List;

import io.smallrye.jdeparser.Expr;
import io.smallrye.jdeparser.JDeparser;
import io.smallrye.jdeparser.Sources;
import io.smallrye.jdeparser.SourceVersion;
import io.smallrye.jdeparser.Type;
import io.smallrye.jdeparser.format.Filer;
import io.smallrye.jdeparser.format.FormatPreferences;

public class EntityGenerator {

    record FieldDef(String name, Type type) {}
    record EntityDef(String packageName, String className, List<FieldDef> fields) {}

    public static void main(String[] args) throws IOException {
        Path outputDir = Path.of(args.length > 0 ? args[0] : "generated-sources");

        // Define entities to generate
        List<EntityDef> entities = List.of(
            new EntityDef("com.example.model", "User", List.of(
                new FieldDef("id", Type.LONG),
                new FieldDef("username", Type.STRING),
                new FieldDef("email", Type.STRING),
                new FieldDef("active", Type.BOOLEAN)
            )),
            new EntityDef("com.example.model", "Product", List.of(
                new FieldDef("id", Type.LONG),
                new FieldDef("name", Type.STRING),
                new FieldDef("price", Type.DOUBLE),
                new FieldDef("quantity", Type.INT)
            ))
        );

        Filer filer = Filer.newInstance(outputDir);
        Sources sources = JDeparser.createSources(
            filer, FormatPreferences.defaults(), SourceVersion.JAVA_17
        );

        for (EntityDef entity : entities) {
            generateEntity(sources, entity);
        }

        sources.writeSources();
        System.out.println("Generated " + entities.size()
            + " entity classes in " + outputDir);
    }

    private static void generateEntity(Sources sources, EntityDef entity) {
        sources.createSourceFile(entity.packageName(), entity.className(), sf -> {
            sf.import_(Type.named("java.util.Objects"));

            sf.class_(entity.className(), cc -> {
                cc.public_();

                cc.docComment(doc -> {
                    doc.text("Generated entity class for " + entity.className() + ".");
                    doc.since("1.0");
                });

                // Fields
                for (FieldDef field : entity.fields()) {
                    cc.field(field.name(), fc -> {
                        fc.private_();
                        fc.type(field.type());
                    });
                }

                // No-arg constructor
                cc.constructor(con -> {
                    con.public_();
                    con.docComment(doc ->
                        doc.text("Creates a new empty " + entity.className() + "."));
                    con.body(body -> {});
                });

                // All-args constructor
                cc.constructor(con -> {
                    con.public_();
                    con.docComment(doc ->
                        doc.text("Creates a new " + entity.className()
                            + " with all fields."));
                    for (FieldDef field : entity.fields()) {
                        con.param(field.name(), field.type(), pc -> {
                            pc.docComment(doc ->
                                doc.text("the " + field.name()));
                        });
                    }
                    con.body(body -> {
                        for (FieldDef field : entity.fields()) {
                            body.emit(Expr.THIS.field(field.name())
                                .assign(Expr.$v(field.name())));
                        }
                    });
                });

                // Getters and setters
                for (FieldDef field : entity.fields()) {
                    String capName = field.name().substring(0, 1).toUpperCase()
                        + field.name().substring(1);

                    // Getter
                    cc.method("get" + capName, mc -> {
                        mc.public_();
                        mc.returning(field.type());
                        mc.docComment(doc ->
                            doc.returnInline("the " + field.name()));
                        mc.body(body -> {
                            body.return_(Expr.$v(field.name()));
                        });
                    });

                    // Setter
                    cc.method("set" + capName, mc -> {
                        mc.public_();
                        mc.returning(Type.VOID);
                        mc.param(field.name(), field.type(), pc -> {
                            pc.docComment(doc ->
                                doc.text("the " + field.name() + " to set"));
                        });
                        mc.docComment(doc ->
                            doc.text("Sets the " + field.name() + "."));
                        mc.body(body -> {
                            body.emit(Expr.THIS.field(field.name())
                                .assign(Expr.$v(field.name())));
                        });
                    });
                }

                // toString
                cc.method("toString", mc -> {
                    mc.public_();
                    mc.returning(Type.STRING);
                    mc.annotate(Type.named("java.lang.Override"));
                    mc.body(body -> {
                        Expr result = Expr.str(entity.className() + "{");
                        boolean first = true;
                        for (FieldDef field : entity.fields()) {
                            if (!first) {
                                result = result.add(Expr.str(", "));
                            }
                            result = result.add(Expr.str(field.name() + "="))
                                .add(Expr.$v(field.name()));
                            first = false;
                        }
                        result = result.add(Expr.str("}"));
                        body.return_(result);
                    });
                });
            });
        });
    }
}
